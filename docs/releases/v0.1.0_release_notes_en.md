# invoice-check-cli v0.1.0

**Release Date**: 2026-05-10
**Pairs with**: AgentD `>= v0.4.9`

First standalone release of the invoice QR-code verification CLI service. Drives an Android device over ADB to verify printed-invoice QR codes via the WeChat 财政票据 service account, then aggregates results into structured artifacts.

---

## Capabilities

- `invoice-check-cli health` — environment check (ADB / VLM endpoint / Paddle endpoint reachability)
- `invoice-check-cli verify-one` — verify a single invoice (by image or already-decoded QR payload)
- `invoice-check-cli verify-batch` — verify a batch from an Excel input

Outputs (per batch run):

- `发票核验结果清单.xlsx` — normalized result spreadsheet
- `validation_screenshots/` — per-invoice on-device verification capture
- `invoice_check_results.zip` — packaged bundle for downstream sharing
- `trace.jsonl`, `summary.json`, `batch_report.json` — internal audit / diagnostics

## Stack

- ADB-driven Android automation (manual login + WeChat session retained on device)
- VLM-assisted on-device UI state recognition (default `Qwen3.6-35B`-class endpoint)
- Optional Paddle OCR/VL endpoint for invoice text re-read

## Server prerequisites

- Reachable Android device via USB or TCP `adb`
- Phone unlocked, WeChat logged in, 财政票据 service account already accessible
- Reachable VLM endpoint (set `INVOICE_CHECK_VLM_API` in `.env`)
- Optional Paddle OCR/VL endpoint (`INVOICE_CHECK_PADDLE_VL_API`)
- Daily query-quota limits on the 财政票据 side — verify on small batches first

## Install (server)

```bash
git clone https://github.com/edisonhsu9919/invoice-check-cli.git \
  /opt/agentd-services/invoice-check-cli/releases/0.1.0
cd /opt/agentd-services/invoice-check-cli/releases/0.1.0
python3 -m venv .venv
.venv/bin/pip install -r requirements.txt

cp config/defaults.json /opt/agentd-services/invoice-check-cli/shared/defaults.json
cp .env.example /opt/agentd-services/invoice-check-cli/shared/.env
# edit shared/.env: VLM endpoint, ADB path, Paddle endpoint (optional)

ln -sfn /opt/agentd-services/invoice-check-cli/shared/.env  releases/0.1.0/.env
ln -sfn /opt/agentd-services/invoice-check-cli/releases/0.1.0  /opt/agentd-services/invoice-check-cli/current

bin/invoice-check-cli health
```

## Behavior contract

- `stdout` emits exactly one final JSON payload on success
- `stderr` streams progress / diagnostics (consumed by AgentD Task Output)
- Workspace-relative absolute paths — same convention as other AgentD CLI services

## Known Non-Blocking Observations

- Reliability depends on phone state (unlocked, WeChat session valid, 财政票据 mini-program reachable). The VLM verifies UI state before each step, but a phone left locked overnight will fail predictably.
- Daily verification-count limits exist on the 财政票据 side; large batches may saturate quota.
- Default VLM endpoint in `.env.example` and `config/defaults.json` is `127.0.0.1:8081` — replace with your real internal endpoint at deploy time.
